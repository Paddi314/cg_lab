1. 实验目标
通过本次实验，你将能够：
1. 深入理解 3D 空间中的坐标变换流程（模型-视图-投影/MVP(Model-View-Projection)变换）。
2. 独立推导并用代码实现模型变换（Model）、视图变换（View）和投影变换（Projection）矩阵。
3. 掌握面向数据编程框架 Taichi 的基本语法与矩阵操作。
2. 实验背景
在提供的代码框架中，我们定义了一个三维空间中的三角形。该三角形的初始顶点坐标为：
$$v_0$$: (2.0, 0.0, -2.0)
$$v_1$$: (0.0, 2.0, -2.0)
$$v_2$$: (-2.0, 0.0, -2.0)
你需要将这三个三维空间中的点，通过 MVP 矩阵 变换为二维屏幕坐标，并在 Taichi 的 GUI 窗口中绘制出对应的线框三角形。
3. 实验要求
你需要补全代码框架中的三个核心函数，返回对应的 $$4 \times 4$$ 齐次坐标变换矩阵：
1. get_model_matrix(angle):
  - 功能: 接收一个旋转角度（角度制），返回绕 Z 轴旋转该角度的模型变换矩阵。
2. get_view_matrix(eye_pos):
  - 功能: 接收相机位置（三维向量），返回视图变换矩阵。你需要将相机平移至世界坐标系的原点。
3. get_projection_matrix(eye_fov, aspect_ratio, zNear, zFar):
  - 功能: 接收视场角（Y 轴方向，角度制）、屏幕长宽比、近截面距离和远截面距离，返回透视投影矩阵。
  - 流程: 先将透视平截头体挤压为正交长方体（透视到正交矩阵$$M_{persp \to ortho}$$），再进行正交投影（$$M_{ortho}$$）。
4. 实验提示
- 角度与弧度转换：Python 中的三角函数（如 ti.sin, ti.tan）使用弧度制。请务必先将传入的 angle 和 eye_fov 乘以 $$\frac{\pi}{180}$$ 转换为弧度。
- 透视投影的边界计算：在构建正交投影矩阵时，你需要计算视锥体的上 (t)、下 (b)、左 (l)、右 (r) 边界。利用视场角（$$fov$$）和近截面绝对距离（$$|n|$$），可以推导：
$$t = \tan(\frac{fov}{2}) \cdot |n|$$
$$b = -t$$
$$r = aspect\_ratio \cdot t$$
$$l = -r$$
- 注意 Z 轴的符号：按照常规的右手坐标系习惯，相机看向 -Z 方向，因此传入的 zNear 和 zFar 虽然是正值距离（例如 0.1 和 50），但在实际坐标中，近截面 $$n = -zNear$$，远截面 $$f = -zFar$$。
- 矩阵乘法顺序：在对顶点 $$\mathbf{v}$$ 应用变换时，由于我们使用的是列向量，矩阵乘法应当遵循右乘规则（即从右向左执行）：$$MVP = M_{proj} @ M_{view} @ M_{model}$$。
- 透视除法：经过 MVP 矩阵变换后，顶点会变成齐次坐标 $$(x, y, z, w)$$。在映射到屏幕之前，必须将 $$x, y, z$$ 分别除以 $$w$$，使其归一化到 $$[-1, 1]$$ 的标准设备坐标系 (NDC) 中。

- 构建 3D 立方体并进行透视旋转：
  目前我们的程序中只绘制了一个扁平的三角形。为了真正展现视图变换的三维空间感，大家可以尝试将二维的三角形升级为真正的三维立体几何体。
7.2 提示
- 构建三维几何体：在代码中定义一个三维正方体（Cube）。正方体有 8 个顶点和 12 条边，请将其中心放置在原点 (0, 0, 0)，边长为 2（即顶点坐标在 [-1, 1] 之间）。
- 修改渲染逻辑：将原本用于三角形的循环绘制逻辑，修改为遍历正方体的 12 条边进行线框绘制。
